# 项目架构说明

## 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   Streamlit 前端                      │
│                    (frontend/)                       │
├─────────────────────────────────────────────────────┤
│                  Agent 编排层 (LangGraph)             │
│                   (agents/)                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 数据分析   │  │ 知识问答   │  │ 报表生成   │          │
│  │ Agent    │  │ Agent    │  │ Agent    │          │
│  └──────────┘  └──────────┘  └──────────┘          │
├─────────────────────────────────────────────────────┤
│              工作流引擎 (workflows/)                   │
├──────────────┬──────────────┬───────────────────────┤
│   RAG 检索    │   数据库层    │    Prompt 模板         │
│   (rag/)     │  (database/) │    (prompts/)         │
├──────────────┴──────────────┴───────────────────────┤
│            工具层 / 通用组件 (utils/)                  │
├─────────────────────────────────────────────────────┤
│         配置管理 / 环境变量 (config/)                   │
└─────────────────────────────────────────────────────┘
```

## 模块职责

### frontend/ — 前端界面
- Streamlit 应用入口
- 用户交互界面
- 图表展示与报表导出

### agents/ — AI Agent 模块
- 多 Agent 定义（数据分析、知识问答、报表生成）
- Agent 之间的协作与编排
- 基于 LangGraph 的状态机管理

### rag/ — RAG 检索增强
- 文档加载与分块
- 向量化存储（ChromaDB）
- 语义检索与重排序

### database/ — 数据库层
- DuckDB 连接管理
- SQL 查询执行
- 数据源适配器

### reports/ — 报表生成
- 数据分析报告模板
- 自动生成与导出
- 可视化图表集成

### workflows/ — 工作流定义
- LangGraph 图定义
- 业务流程编排
- 状态传递与条件分支

### prompts/ — Prompt 模板
- 系统提示词管理
- 场景化模板
- 版本控制

### utils/ — 工具函数
- 日志管理
- 数据格式化
- 通用辅助函数

### config/ — 配置管理
- 环境变量加载
- 应用配置
- 多环境支持

## 核心数据流

```
用户输入 → Streamlit 前端
    ↓
LangGraph 工作流调度
    ↓
┌─────────┬──────────┬──────────┐
│ RAG检索  │ 数据库查询 │ Agent推理  │
└─────────┴──────────┴──────────┘
    ↓
结果整合 → 前端展示 / 报表导出
```

## 技术选型理由

- **LangGraph**：提供灵活的 Agent 编排，支持循环、条件分支和状态管理
- **ChromaDB**：轻量级向量数据库，适合本地开发和中小规模部署
- **DuckDB**：嵌入式分析数据库，无需独立服务，性能优异
- **Streamlit**：快速构建数据应用的 Python 框架，适合内部工具开发
