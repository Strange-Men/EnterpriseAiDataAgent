# M4-8 Frontend Skeleton Lock

## 概述

本文档锁定 M4-8.x 系列的前端重构骨架，定义页面结构、重构顺序和改动上限，避免边写边乱改。

---

## 1. 页面骨架

```
Home (/)
  └── 产品概览 + 快速入口 + 最近活动

Data (/data)
  └── 数据表管理 + 上传 + 预览 + 质量检查

Analyze (/analyze)
  ├── Natural Language Query (AI 问答)
  └── Expert SQL (SQL 编辑器)

History (/history)
  └── 查询历史 + 筛选 + 导出

Analysis Detail (/analyze/[id])
  └── 分析报告 + 步骤 + 结果 + 导出

Settings (/settings)
  └── 语言 + 主题 + API 状态
```

---

## 2. 每页核心目标

### 2.1 Home

**目标**：让用户一屏知道产品用途和下一步。

**核心元素**：
- 产品简介（一句话）
- 快速入口卡片（上传数据 / 开始分析 / 查看历史）
- 最近活动摘要（最近 3-5 条查询/分析）
- 系统状态指示（AI 服务 / 数据库）

**不做的事**：
- 不堆砌功能入口
- 不放复杂图表
- 不放教程/引导（那是 onboarding 的事）

### 2.2 Data

**目标**：管理数据表、上传、预览、质量检查。

**核心元素**：
- 数据表列表（名称、行数、列数、大小）
- 上传按钮（CSV/Excel）
- 表详情（预览、Schema、质量报告）
- 删除/重命名操作

**不做的事**：
- 不做复杂的数据清洗
- 不做数据转换
- 不做多表关联

### 2.3 Analyze

**目标**：核心工作台，自然语言和专家 SQL。

**核心元素**：
- Tab 切换（Natural Language / Expert SQL）
- SQL 编辑器（Monaco）
- 结果表格
- AI 分析面板（可选）
- Toolbar（执行 / AI / 格式化 / 保存）

**不做的事**：
- 不做复杂的多面板布局
- 不做 schema 侧栏（M4-8.3 考虑）
- 不做可视化图表（未来版本）

### 2.4 History

**目标**：回查、复用、导出。

**核心元素**：
- 查询历史列表（时间、类型、状态）
- 搜索和筛选
- 操作按钮（重新执行 / 加载到工作台 / 导出）
- 清空历史

**不做的事**：
- 不做复杂的分页
- 不做批量操作
- 不做历史分析

### 2.5 Analysis Detail

**目标**：报告阅读、导出、重新运行。

**核心元素**：
- 报告标题 + 摘要
- 分析步骤时间线
- 结果表格
- 导出按钮（Markdown / CSV）
- 重新运行按钮

**不做的事**：
- 不做报告编辑
- 不做报告分享
- 不做报告模板

### 2.6 Settings

**目标**：语言、主题、API 状态。

**核心元素**：
- 语言切换（中文 / English）
- 主题切换（暗色 / 亮色）
- AI 服务状态
- 数据库状态

**不做的事**：
- 不做复杂的配置
- 不做用户管理
- 不做权限设置

## 4. 每轮改动上限

### 4.1 范围限制

- **每轮最多 1-2 个核心页面**
- **不改后端**（API / Store / 数据库）
- **不改 store**，除非是显式状态任务
- **不改业务逻辑**（SQL 执行 / AI 生成 / 数据上传）

### 4.2 质量要求

- **每轮必须有 tsc/test/build**
- **每轮必须有线上人工验证清单**
- **每轮必须有回归检查**（不破坏现有功能）

### 4.3 文件限制

- **优先编辑现有文件**，不创建新文件
- **不创建新的组件目录**，除非是明确的组件提取
- **不创建新的 store**，除非是明确的状态需求

---

## 5. 严格禁止

### 5.1 不恢复实验功能

- 不恢复 Templates / Scheduler / Charts / Anomalies / Diff / Timeline
- 不恢复 `/performance`、`/virtual-table`
- 不恢复 Command Palette / Global Search / Keyboard Shortcuts

### 5.2 不做重型 BI 大屏

- 不把项目改成复杂 BI 仪表盘
- 不做多图表布局
- 不做复杂的筛选器
- 不做数据转换工具

### 5.3 不破坏核心体验

- 不把 Trace 放到主视觉
- 不把 SQL editor 搞得更挤
- 不牺牲功能稳定换视觉效果
- 不改 SQL 执行逻辑
- 不改 AI 生成逻辑

### 5.4 不引入新依赖

- 不引入新的 UI 库（保持 Tailwind + 自定义组件）
- 不引入新的状态管理库（保持 Zustand）
- 不引入新的路由方案（保持 Next.js App Router）

## 8. 总结

M4-8.x 系列是前端 UI/UX 的渐进式改进，不是大规模重构。每轮聚焦 1-2 个页面，遵循：

1. **克制**：不堆砌功能，不滥用颜色
2. **层级**：标题是导航辅助，内容是主体
3. **一致**：使用 design tokens，组件复用
4. **效率**：快捷键完整，操作路径短

严格禁止恢复实验功能、引入新依赖、破坏核心体验。每轮必须有 tsc/test/build 验证。
