## 2. 值得借鉴的 UI/UX 点

### 2.1 暗色主题设计

**Linear 风格克制暗色**：
- 背景色层次：`#0E1117` → `#161B22` → `#21262D`（已采用类似方案）
- 文字色层级：主文字不用纯白，用 `#E6EDF3` 或更低
- 强调色克制：仅用于交互元素，不用于大面积装饰
- 边框极细：`1px solid rgba(255,255,255,0.06)` 或类似
- 阴影极轻：避免重阴影，用微妙的背景色差区分层级

**关键原则**：
- 暗色主题不是"把白色换成黑色"
- 减少视觉噪音，让用户聚焦内容
- 用空间和层级而非颜色区分区域

### 2.2 SQL 编辑器布局

**Retool / Hex 模式**：
- Toolbar 在顶部，按钮分组（执行相关 / AI 相关 / 格式化 / 保存）
- Editor 占主要空间，有足够 padding（至少 12px）
- 结果表在下方，可折叠/调整大小
- Schema 浏览器在侧栏（可选），不挤占编辑器空间

**关键原则**：
- 编辑器是核心，周围元素为其服务
- 行高至少 20px，字号至少 13px
- 行号区域有足够宽度（至少 40px）
- Tab bar 在编辑器上方，不与 toolbar 混淆

### 2.3 分析结果展示

**Hex / Julius 模式**：
- 结果表与编辑器同宽，不缩进
- 列宽自适应，长文本可展开
- 分页或虚拟滚动，不一次性加载全部
- 结果上方有统计行（行数、耗时）
- 错误信息用红色背景块，不与结果混排

**关键原则**：
- 结果是编辑器的延伸，不是独立区域
- 信息密度适中，不挤不空
- 状态指示清晰（成功/错误/加载中）

### 2.4 页面标题与层级

**Linear / Vercel 模式**：
- 页面标题用 `text-secondary` 而非 `text-primary`
- 副标题用 `text-muted`，更小字号
- 区块标题用 `uppercase tracking-wider` 样式
- 标题与内容之间有合理间距（16-24px）

**关键原则**：
- 标题是导航辅助，不是视觉焦点
- 内容才是页面主体
- 层级通过字号、字重、颜色三个维度体现

### 2.5 导航与信息架构

**Arc / Linear 模式**：
- 侧栏导航简洁，图标 + 文字
- 当前页面高亮，但不用强色
- 面包屑用于深层页面
- 快捷键提示内嵌在菜单中

**关键原则**：
- 导航不抢内容风头
- 用户随时知道自己在哪
- 返回路径清晰

## 3. 不适合本项目照搬的点

### 3.1 重型 BI 仪表盘

- **Tableau / PowerBI 风格**：多图表、多筛选器、复杂布局
- **原因**：本项目定位是轻量 AI 数据分析工作台，不是 BI 大屏
- **适用**：保持单表分析、单 SQL 执行的简洁流程

### 3.2 复杂的多面板布局

- **Databricks 风格**：左侧目录树、中间编辑器、右侧属性面板、底部结果
- **原因**：当前功能不需要那么多面板，强行加会增加认知负担
- **适用**：保持当前的单列布局，用 tab 切换不同视图

### 3.3 过度的动效设计

- **某些 SaaS 产品**：页面转场动画、元素入场动效、微交互
- **原因**：数据分析工具需要效率，动效会拖慢操作节奏
- **适用**：仅在状态变化时用轻量 transition（150-200ms）

### 3.4 社交/协作功能

- **Hex / Notion 风格**：多人协作、评论、@提及
- **原因**：当前版本是单用户工具，不需要协作功能
- **适用**：保持单用户工作流，未来可扩展

## 4. 对本项目的具体启发

### 4.1 SQL 编辑器改进方向

| 改进项 | 当前状态 | 目标状态 | 优先级 |
|--------|---------|---------|--------|
| 编辑器 padding | 8px | 12px | 高 |
| 行高 | 默认 | 20px | 高 |
| 最小高度 | 200px | 280px | 高 |
| Toolbar 分组 | 平铺 | 执行/AI/格式化 分组 | 中 |
| 结果表与编辑器间距 | 8px | 12px | 中 |
| Schema 侧栏 | 无 | 可选折叠侧栏 | 低 |

### 4.2 页面标题层级

| 改进项 | 当前状态 | 目标状态 | 优先级 |
|--------|---------|---------|--------|
| PageHeader 标题颜色 | text-primary | text-secondary | 高 |
| PageHeader 标题字重 | font-bold | font-semibold | 高 |
| 区块标题样式 | 普通文字 | uppercase tracking-wider | 中 |
| 标题与内容间距 | 6px (mb-6) | 4px (mb-4) | 中 |

### 4.3 暗色主题优化

| 改进项 | 当前状态 | 目标状态 | 优先级 |
|--------|---------|---------|--------|
| 主文字色 | #E6EDF3 | #C9D1D9（降一级） | 中 |
| 边框色 | #30363D | 更微妙 | 低 |
| 卡片背景 | #161B22 | 保持 | - |
| 强调色使用 | 适中 | 保持克制 | - |

### 4.4 信息密度

| 改进项 | 当前状态 | 目标状态 | 优先级 |
|--------|---------|---------|--------|
| 页面 padding | 6px (px-6) | 保持 | - |
| 卡片内 padding | 适中 | 保持 | - |
| 列表项间距 | 8px (space-y-2) | 保持 | - |
| 按钮间距 | 8px (gap-2) | 保持 | - |

## 5. 对 M4-8.2 ~ M4-8.5 的影响

### 5.1 M4-8.2 Home + Navigation Clarity

**参考 Linear 的首页设计**：
- 一屏展示产品用途和下一步操作
- 快速入口卡片（上传数据 / 开始分析 / 查看历史）
- 最近活动摘要
- 不要堆砌功能，保持简洁

**参考 Vercel 的导航设计**：
- 侧栏导航保持简洁
- 当前页面用微妙高亮
- 面包屑用于分析详情页

### 5.2 M4-8.3 Analysis Workspace UX Polish

**参考 Hex 的工作台布局**：
- SQL 编辑器是核心，周围元素为其服务
- 结果表与编辑器同宽
- AI 功能作为辅助，不抢主视觉

**参考 Retool 的 toolbar 设计**：
- 按钮分组：执行相关 / AI 相关 / 格式化 / 保存
- 快捷键提示内嵌
- 状态指示清晰

### 5.3 M4-8.4 Analysis Detail Report Layout

**参考 Mode Analytics 的报告设计**：
- 报告标题 + 摘要在顶部
- 分析步骤按时间线排列
- 结果表格内嵌在步骤中
- 导出按钮在右上角

**参考 Notion 的块级布局**：
- 每个分析步骤是一个"块"
- 块之间有清晰分隔
- 可折叠/展开详情

### 5.4 M4-8.5 History UX Polish

**参考 Linear 的列表设计**：
- 列表项有清晰的视觉层级
- 状态指示用小圆点
- 操作按钮 hover 才显示
- 搜索和筛选在顶部

**参考 Vercel 的卡片设计**：
- 卡片边框极细
- hover 时边框变色
- 内容密度适中

## 6. 设计原则总结

### 6.1 克制原则

- 暗色主题不是"把白色换成黑色"
- 强调色仅用于交互元素
- 动效仅在状态变化时使用
- 信息密度适中，不挤不空

### 6.2 层级原则

- 页面标题是导航辅助，不是视觉焦点
- 内容才是页面主体
- 层级通过字号、字重、颜色三个维度体现
- 区块标题用 uppercase tracking-wider 样式

### 6.3 一致性原则

- 同类元素使用相同样式
- 间距使用统一的 spacing scale
- 颜色使用 design tokens
- 组件复用，不重复造轮子

### 6.4 效率原则

- 快捷键体系完整
- 操作路径最短
- 状态反馈即时
- 错误信息清晰

## 8. 总结

本项目定位是**轻量 AI 数据分析工作台**，不是重型 BI 仪表盘。UI/UX 改进应遵循：

1. **克制**：不堆砌功能，不滥用颜色，不过度动效
2. **层级**：标题是导航辅助，内容是主体，层级清晰
3. **一致**：使用 design tokens，组件复用，间距统一
4. **效率**：快捷键完整，操作路径短，反馈即时

后续 M4-8.2 ~ M4-8.5 将按此原则渐进式改进，每轮最多 1-2 个核心页面，不破坏现有功能。
