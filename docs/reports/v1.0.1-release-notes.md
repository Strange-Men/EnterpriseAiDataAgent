# v1.0.1 — Architecture Split, AI Quality Gates, Performance Hardening

> Date: 2026-06-03
> Scope: v1.0.x architecture optimization pass

## 架构拆分

- `frontend-react/src/services/api.ts` 拆分为 `http-client/status/tables/query/data/ai/streams/envelope` feature modules，旧入口继续兼容 re-export。
- `ai-analysis-panel.tsx` 拆出 mode handlers、loading skeleton、suggested questions 等组件。
- `sql-workspace-panel.tsx` 拆出 workflow banner、AI SQL input、query tabs、query stats 等组件。
- 新增后端 `backend/models/envelope.py` 和前端 envelope 类型，为统一 API / AI result contract 做基础。

## AI 分析有效性

- 新增 `backend/utils/llm_sql.py`，把 LLM 原始输出规整为可执行只读 SQL 或明确 `-- CANNOT_ANSWER`。
- `generate_sql()` 接入 SQL 提取、只读校验和 `quality_gates`。
- SQL generation prompt 改为 `DATABASE_SCHEMA` / `QUESTION` 块格式，减少模型误判问题缺失。
- `/ai/query` 透传 SQL generation quality gates。
- 前端 SQL Workspace 新增 AI SQL quality gates 可视化反馈。
- 新增 `tests/test_llm_sql.py`。
- Golden evaluation 默认跳过真实模型调用，新增 `tests/ai/generate_golden_report.py` 生成可选真实评估报告。

## 性能硬化

- SQL Workspace 默认查询分页从 10000 行调整为 1000 行。
- DataTable 使用 `useDeferredValue()` 延后大表过滤计算。
- AI streaming 解释使用 `requestAnimationFrame` 批量更新，减少高频 setState。

## 数据治理

- 新增 `scripts/migrate-duckdb-path.py`，非破坏性迁移误放在 `backend/data/` 的 DuckDB 文件。
- 新增 `scripts/repository-health-check.py`，检查 `.env`、缓存、误放 DB、`__pycache__` 等仓库健康问题。
