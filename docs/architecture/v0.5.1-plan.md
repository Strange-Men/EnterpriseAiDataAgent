## Module 1: AI Streaming Response (SSE)

### 目标
AI explain/insights 实时输出，不再等待完整响应。

### 风险评估
- **风险**: 中 — 需要改 4 层（LLM调用 → 服务层 → 路由 → 前端）
- **数据流影响**: 新增 SSE channel，不影响现有 JSON 接口（保留向后兼容）
- **耦合**: 不增加 store 耦合，流式数据走 EventSource/fetch stream
- **主工作流影响**: 正面 — 用户看到 AI "打字"效果，体验显著提升
- **God Store 风险**: 无 — 流式状态是组件级 useState，不入 store

### 后端变更

#### 1a. `backend/services/ai_analyst.py` — 新增流式 LLM 调用

```python
# 新增函数，不修改现有 _call_llm
async def _call_llm_stream(system: str, user_message: str, max_tokens: int = 1024) -> AsyncGenerator[str, None]:
    """Yield text chunks as they arrive from Anthropic streaming API."""
    client = _get_client()
    async for event in client.messages.stream(
        model=DEFAULT_LLM_MODEL,
        max_tokens=max_tokens,
        system=system,
        messages=[{"role": "user", "content": user_message}],
    ):
        if event.type == "content_block_delta":
            yield event.delta.text
```

注意：anthropic SDK 的 streaming 是 `client.messages.stream()` 返回 context manager + iterator。需要确认 SDK 版本支持。如果 SDK 不支持 async stream，则用 sync stream + asyncio.to_thread 包装。

#### 1b. `backend/routes/ai.py` — 新增 SSE 端点

```python
from fastapi.responses import StreamingResponse
import json

@router.post("/ai/explain/stream")
async def ai_explain_stream(req: ExplainRequest):
    """SSE stream of AI explanation."""
    async def event_generator():
        async for chunk in _call_llm_stream(EXPLANATION_SYSTEM, ...):
            yield f"data: {json.dumps({'type': 'text', 'content': chunk})}\n\n"
        yield f"data: {json.dumps({'type': 'done'})}\n\n"
    return StreamingResponse(event_generator(), media_type="text/event-stream")
```

保留现有 `/ai/explain` 非流式端点不变。

新增端点（仅对 explain 和 insights 做流式，SQL generation 和 chart-suggest 保持批量）：
- `POST /ai/explain/stream` — 流式 explain
- `POST /ai/insights/stream` — 流式 insights（流式输出后前端自行解析 JSON）

#### 1c. 前端 API 层 — `services/api.ts`

```typescript
// 新增流式调用函数
export function streamAiExplain(
  question: string, sql: string, results: Record<string, unknown>[],
  onChunk: (text: string) => void,
  onDone: () => void,
  onError: (err: Error) => void,
): AbortController {
  const controller = new AbortController();
  fetch(`${API_BASE}/ai/explain/stream`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ question, sql, results }),
    signal: controller.signal,
  }).then(async (res) => {
    const reader = res.body!.getReader();
    const decoder = new TextDecoder();
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      const text = decoder.decode(value);
      for (const line of text.split("\n")) {
        if (line.startsWith("data: ")) {
          const data = JSON.parse(line.slice(6));
          if (data.type === "text") onChunk(data.content);
          if (data.type === "done") onDone();
        }
      }
    }
  }).catch(onError);
  return controller;
}
```

#### 1d. 前端渲染 — `ai-analysis-panel.tsx`

在 explain/insights 模式下，使用流式 API 替代批量 API：
- 新增 `streamingContent` state（string，逐 chunk 追加）
- `ReactMarkdown` 渲染 `streamingContent`（会实时更新）
- 保留 loading skeleton 用于初始状态
- 批量 API（非流式）作为 fallback 保留

### 测试
- 后端：测试 SSE 端点返回正确 Content-Type 和 data: 前缀格式
- 前端：测试 streamAiExplain 正确解析 SSE 行

## Module 2: AI Session Memory

### 目标
AI 能引用之前的分析结果，支持多轮对话式分析。

### 风险评估
- **风险**: 低 — 新增 store + 修改 prompt 注入逻辑
- **数据流影响**: 新增 `ai-session-store.ts`，读取已有 queryResult
- **耦合**: session store 读取 sql-workspace-store 的 queryResult（单向读取，不写入）
- **主工作流影响**: 正面 — follow-up 问题变得可能
- **God Store 风险**: **需严格约束** — 只存对话历史和上下文引用，不存 query result 副本

### 新增文件

#### 2a. `frontend-react/src/stores/ai-session-store.ts`

```typescript
interface AiTurn {
  id: string;
  role: "user" | "assistant";
  content: string;
  sql?: string;
  timestamp: string;
}

interface AiSessionState {
  // 对话历史（用于 UI 渲染 + prompt 注入）
  turns: AiTurn[];

  // 上下文引用（不存数据副本，只存元信息）
  activeTable: string | null;
  lastSql: string | null;
  lastColumns: string[] | null;
  lastRowCount: number | null;

  // Actions
  addUserTurn: (content: string) => void;
  addAssistantTurn: (content: string, sql?: string) => void;
  setContext: (ctx: { table?: string; sql?: string; columns?: string[]; rowCount?: number }) => void;
  clear: () => void;
}
```

**关键设计**: 不存 query result 数据本身。数据仍在 sql-workspace-store.queryResult。session store 只存元信息（columns, rowCount）和对话历史文本。需要完整结果时，从 sql-workspace-store 读取。

#### 2b. `backend/services/ai_analyst.py` — 支持 conversation history 注入

```python
def explain_results(
    question: str, sql: str, results: list[dict],
    conversation_history: list[dict] | None = None  # 新增
) -> dict:
    # 在 user_message 前面拼接历史
    history_context = ""
    if conversation_history:
        for turn in conversation_history[-6:]:  # 最多保留最近 6 轮
            history_context += f"\n[{turn['role']}]: {turn['content']}"
    user_message = f"""Previous conversation:{history_context}
Current question: {question}
SQL executed: {sql}
Results (first 50 rows): {json.dumps(results[:50], default=str)}
"""
```

#### 2c. `backend/routes/ai.py` — 请求模型新增可选字段

```python
class ExplainRequest(BaseModel):
    question: str
    sql: str
    results: list[dict]
    conversation_history: list[dict] | None = None  # 新增
```

### 测试

- 后端：带 conversation_history 的 explain 返回更相关的解释

## Module 3: Result-aware Follow-up Analysis

### 目标
查询执行后，AI 自动感知结果 schema/sample，用户可直接追问。

### 风险评估
- **风险**: 中 — 需要串联 Module 1 + 2，修改 SqlWorkspacePanel 的 AI 按钮行为
- **数据流影响**: AI panel 现在读取 session store + sql workspace store
- **耦合**: AI panel 读取两个 store（session + workspace），单向读取
- **主工作流影响**: 核心增强 — "为什么利润下降？" → "退款来自哪些产品？" 变得可行
- **God Store 风险**: 无 — follow-up 逻辑在组件层，不在 store

### 前端变更

#### 3a. `ai-analysis-panel.tsx` — 新增 follow-up 输入

在分析结果下方新增一个文本输入框：
```
[  继续追问...  ] [发送]
```

发送时：
1. 将用户问题加入 `aiSessionStore.addUserTurn(question)`
2. 从 `sqlWorkspaceStore.queryResult` 读取当前结果
3. 从 `aiSessionStore` 读取 `turns` 作为 conversation_history
4. 调用流式 explain API（复用 Module 1）
5. 回答完成后 `aiSessionStore.addAssistantTurn(response, sql)`

#### 3b. `sql-workspace-panel.tsx` — AI 按钮行为增强

当点击 "AI Explain" 时：
1. 检查 aiSessionStore 是否有历史
2. 如果有，将历史作为 conversation_history 传入
3. AI panel 打开时自动显示之前的对话轮次

### 后端变更

#### 3c. `ai_pipeline.py` — follow-up SQL 生成

## Module 5: 主链路串联 + 测试验证

### 目标
将 Module 1-4 串联进主工作流，确保端到端可演示。

### 主工作流增强版本

```
Upload CSV
  → Auto Analyze (full-analysis，已有)
  → AI Summary (流式输出 ← Module 1)
  → AI Generate SQL (已有)
  → Open in SQL Workspace (已有)
  → Execute Query (已有)
  → AI Explain Result (流式 ← Module 1, 带上下文 ← Module 2)
  → User Follow-up: "为什么？" (← Module 3)
  → AI answers with context (← Module 2+3)
  → Render Charts (← Module 4)
  → Export
```

### workflow-store 增强（极小改动）

不改 workflow-store 结构，只在现有 stage 基础上：
- `"analyzing"` 阶段触发流式 AI summary
- `"done"` 阶段可从 AI panel 发起 follow-up（不改 stage，因为 follow-up 是 done 阶段的子交互）
