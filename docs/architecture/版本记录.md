## v0.7.2 — AI Session Quality Layer (2026-05-25)

- 写入时同步到 DuckDB，自动清理旧条目
- 前端 `sql-history-store.ts` 添加 `fetchHistory()` 从后端读取历史

### 大数据集稳定性
- `db_manager.py` 添加 `timeout_ms` 参数，设置 `statement_timeout`
- `query_executor.py` 超时时返回友好错误信息（提示添加 LIMIT）
- COUNT 查询使用较短超时（60s）

### Query Memory/Cache Protection
- `sql-workspace-store.ts` 添加 queryResult 大小守卫（50MB 警告）
- `sql-history-store.ts` 添加 localStorage 大小守卫（2MB 限制）
  - 截断长 SQL（>1000 字符）
  - 截断长 error（>500 字符）
  - 超限时自动丢弃最旧条目
- `analysis-store.ts` 优化压缩逻辑
  - 保存和未保存 runs 都压缩
  - Chart data 限制 100 点
  - Step data 限制 50 行
  - Trace events 限制 20 条
  - 4MB 超限时优先丢弃最旧 runs

### Long-Session Reliability
- `api.ts` SSE 流添加自动重连机制
  - 最多 2 次重试
  - 指数退避（1s, 2s）
  - 仅在未收到数据时重试
- `consumeSseStream` 和 `consumeSseStreamGeneric` 都支持重连
