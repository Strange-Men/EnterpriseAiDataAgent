# 版本记录 — Enterprise AI Data Agent

## v0.5.4 — AI System Engineering: Prompt Architecture, Token Budget, Guardrails, Trace, Evaluation (2026-05-24)

### Module 1: Prompt Architecture Layer (已完成)
- 创建 `backend/prompts/` 目录，8 个 prompt 模块
- 定义 `PromptContract` 数据类：name, purpose, required/optional vars, response_format, max_output_tokens
- 每个模块导出：CONTRACT, SYSTEM_PROMPT, build_user_message()
- `registry.py` 集中注册所有 prompt 合约
- `locale.py` 统一语言处理（从 ai_analyst.py 迁移）
- `ai_analyst.py` 所有 prompt 常量和 user message 构建逻辑迁移至 prompts 层
- `ai_pipeline.py` 移除 `_SUMMARY_SYSTEM`，改为导入 `summarizer` 模块
- 服务层零 prompt 文本，全部从 prompts 导入
- 新增 skill: `prompt-architecture.md`

### Module 2: Token Budget System (已完成)
- 创建 `backend/runtime/token_budget.py`
- `TokenBudget` 数据类定义 per-operation 预算（input/output/schema/sample/history/workflow_total）
- `estimate_tokens()` 启发式估算（len//3），禁止第三方 tokenizer
- `truncate_text()` / `truncate_rows()` 截断工具
- `WorkflowTokenTracker` 多步骤 workflow token 追踪，25K 总预算
- 8 类 operation 预算配置：sql_gen(3K), explain(6K), insights(5K), charts(3K), semantics(3K), questions(2K), plan(3K), summarizer(3K)
- `_call_llm` / `_call_llm_stream` 集成 budget enforcement + tracker recording
- 所有 public 函数使用 budget 配置替代硬编码值（行数、max_tokens）
- autonomous analysis 集成 tracker: 超预算跳过剩余步骤
- 响应新增 `token_budget` 字段
- 新增 skill: `token-budget-control.md`

### Module 3: Runtime Guardrails (已完成)
- 创建 `backend/services/guardrails.py`
- `AnalysisGuardrails` 数据类：max_steps(6), max_sql_queries(8), max_consecutive_failures(2), max_total_time(120s), max_step_time(30s), max_recursion_depth(2), require_minimum_success(1)
- `GuardrailViolation` 异常类，携带 rule + message + context
- `AnalysisGuard` 类：check_before_step(), record_step_result(), check_after_all(), to_dict()
- 预设 STRICT 模式：max_steps=4, max_sql=5, consecutive_fail=1, timeout=60s
- 循环依赖检测：`_chain_depth()` 追踪 depends_on 链
- `run_autonomous_analysis` 集成：每步前检查 guardrail，违规时跳过或停止
- `run_autonomous_analysis_stream` 同步集成
- 响应新增 `guardrails` 和 `guardrail_violations` 字段
- `MultiAnalyzeRequest` 新增 optional `guardrails: dict` 参数
- 新增 skill: `runtime-guardrails.md`

### Module 4: Analysis Trace System (已完成)
- 创建 `backend/services/trace.py`
- `TraceEvent` 数据类：timestamp, operation, phase, prompt_name, input/output tokens, latency, status, error, sql, step
- `AnalysisTrace` 数据类：trace_id, question, table, mode, events, plan, guardrail_violations
- `TraceRecorder` 类：record_llm_call(), record_guardrail_violation(), set_plan(), finish(), to_dict(), save_to_file()
- `_call_llm` / `_call_llm_stream` 新增 trace/phase/prompt_name/step 参数，成功/失败均记录
- 所有 public 函数接受 optional trace 参数并传递至 _call_llm
- `run_autonomous_analysis` 创建 TraceRecorder，规划/步骤/摘要阶段全部记录
- `run_autonomous_analysis_stream` 同步集成
- 响应新增 `trace` 字段：trace_id, total_llm_calls, total_input/output tokens, events
- guardrail 违规记录到 trace
- 新增 skill: `analysis-trace.md`
- 旧: 8 个 prompt 散落在 2 个 service 文件，无 contract，无结构化 builder
- 新: 8 个 prompt 集中在 `backend/prompts/`，每个有 contract + builder

## v0.5.1 — AI Streaming, Session Memory, Follow-up Analysis & Chart Rendering (2026-05-24)

### AI Streaming Response (SSE)
- Backend: `_call_llm_stream` — Anthropic SDK streaming via `client.messages.stream()`
- Backend: `POST /ai/explain/stream`, `POST /ai/insights/stream` — SSE endpoints
- Frontend: `consumeSseStream`, `streamAiExplain`, `streamAiInsights` — generic SSE consumer
- AI Analysis Panel: live streaming render with pulsing indicator

### AI Session Memory
- New store: `ai-session-store.ts` — conversation turns (max 20), context metadata
- Fields: `turns`, `activeTable`, `lastColumns`, `lastRowCount`, `lastSql`, `lastInsightSummary`
- Backend: `ExplainRequest.conversation_history` — bounded to 6 turns in prompt
- 11 store tests (initial state, turns, MAX_TURNS cap, context, clear, lastSql/lastInsightSummary)

### Result-aware Follow-up Analysis
- Backend: `build_follow_up_context()` — structured context injection (schema + 5 sample rows + previous SQL + insight summary)
- Backend: `FollowUpContext` Pydantic model, `AIQueryRequest.follow_up_context`
- Token budget: ~220-500 tokens (vs ~50,000+ for full result injection)
- Frontend: follow-up input in AI Analysis Panel → `aiQuery(execute: false)` → user confirms → execute
- Fallback: empty context degrades to normal query flow

### Chart Rendering (Recharts)
- New component: `ai-chart.tsx` — ChartSpec → Recharts (bar/line/pie/scatter)
- AI-driven: `aiChartSuggest` auto-called after explain → charts embedded in analysis panel
- Max 2 charts, non-blocking fallback on failure
- No manual chart type selection — AI decides
