# LLM Provider Configuration

## Default Behavior

Mock LLM is the default and safe fallback. The platform remains runnable when no real provider key is configured.

## Supported Providers

- `mock`
- `deepseek`
- `doubao`
- `mimo`

## Environment Variables

Backend-only variables:

| Variable | Purpose |
| --- | --- |
| `LLM_MODE` | Runtime mode. Default: `mock`. |
| `LLM_DEFAULT_PROVIDER` | Provider used when the request does not specify one. Default: `mock`. |
| `LLM_ALLOWED_PROVIDERS` | Comma-separated provider allowlist. |
| `LLM_FALLBACK_PROVIDER` | Fallback provider. Must remain `mock` for safe demo behavior. |
| `LLM_FALLBACK_ON_ERROR` | Fallback to mock when real provider config or request fails. |
| `LLM_REQUEST_TIMEOUT_SECONDS` | Real provider request timeout. |
| `LLM_MAX_RETRIES` | Real provider retry count before fallback. |
| `DEEPSEEK_API_KEY` / `DEEPSEEK_BASE_URL` / `DEEPSEEK_MODEL` | DeepSeek OpenAI-compatible backend config. |
| `DOUBAO_API_KEY` / `DOUBAO_BASE_URL` / `DOUBAO_MODEL` | Doubao / Volcano Ark backend config. |
| `MIMO_API_KEY` / `MIMO_BASE_URL` / `MIMO_MODEL` | Mimo backend config. |

Frontend public variables:

| Variable | Purpose |
| --- | --- |
| `NEXT_PUBLIC_API_BASE_URL` | Backend API base URL. |
| `NEXT_PUBLIC_API_URL` | Backward-compatible backend URL for existing API clients. |
| `NEXT_PUBLIC_LLM_PROVIDER_SELECTOR_ENABLED` | Enables the provider selector UI. |
| `NEXT_PUBLIC_DEFAULT_LLM_PROVIDER` | Frontend default provider name. |
| `NEXT_PUBLIC_ALLOWED_LLM_PROVIDERS` | Frontend provider names only, no keys. |

## Frontend Selection

Users can select the provider in the Analyze panel header:

- Mock LLM
- DeepSeek
- Doubao
- Mimo

The frontend sends only the provider name as `llm_provider`. It never sends API keys, Authorization headers, or provider secrets.

## Fallback Behavior

If a real provider is selected but key, base URL, or model is missing, the backend returns a normal response generated by Mock LLM and marks the response metadata:

```json
{
  "llm": {
    "mode": "mock",
    "provider_requested": "deepseek",
    "provider_used": "mock",
    "fallback_triggered": true,
    "fallback_reason": "provider_unavailable"
  }
}
```

If a real provider request fails because of an expired key, auth error, rate limit, timeout, provider outage, or malformed response, the backend also falls back to Mock LLM with `fallback_reason: "provider_request_failed"`.

## Security Boundary

- API keys only live in backend environment variables.
- Frontend config only contains provider names and public API URLs.
- No real key is committed.
- No Authorization header is exposed in browser code.
- Error responses and metadata do not include key values or Authorization contents.
